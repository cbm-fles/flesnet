
Three components:
- stsender
- tsbuilder
- tssched

# Draft protocol

stsender (connect) <-> tsbuilder (listen)
----------------------

-> REGISTER
     hostname, pid
<- OK

-> ANNOUNCE_SUBTIMESLICE
     id, size
<- OK

<- RELEASE_SUBTIMESLICE
     id
-> OK

-> UNREGISTER
<- OK

(What happens if one of the partners stops unexpectedly?
Should we do heartbeating?)


tsbuilder <-> tssched
---------------------

-> REGISTER
     hostname, pid
<- OK

-> MEMORY_AVAILABLE
     size
<- OK

<- NEW_TIMESLICE
     serialized descriptor
-> OK

-> RELEASE_TIMESLICE / FAILED_TIMESLICE(?)
     timeslice id
     success state
<- OK

-> UNREGISTER
<- OK


tsbuilder (connect) <-> stsender (listen)
----------------------

-> REQUEST_SUBTIMESLICE
    timeslice id

<- SEND_SUBTIMESLICE
    return status
    serialized descriptor, timeslice data (optional)
